What is the hardest part about starting a new project from scratch?

  • Just getting started
  • So many things to setup (i.e. logging, security, auditing, monitoring, etc)
  • Making all the library decisions

I'm pretty sure I want...

  • A responsive asynchronous front-end
  • REST Services
  • A database/ORM

Enter JHipster

  • A responsive front-end using HTML5/CSS3/JavaScript frameworks
  • A robust and high-quality back-end, with the latest Java/Caching/Data access technologies
  • All automatically wired up, with security and performance in mind
  • Great developer tooling for maximum productivity
  • Open source so if you need something, add it
  • Production-Ready (gzipping, HTTP header caching, minification, ready for cloud)

Things you probably already have:

(if not, you will need these soon)

  • Java (7/8)
  • Maven (or Gradle)
  • Git
  • Node.js

Client-Side Technologies

  • Yeoman - Provides application generators (mostly JavaScript)
  • Grunt - JavaScript task runner (live refresh, compress/optimize/compile)
  • Bower - Package manager for JavaScript assets
  • Karma - Runs JavaScript unit tests (using headless browser)
  • AngularJS - Most popular JavaScript framework
  • Twitter Bootstrap - CSS/JavaScript framework for responsive web apps

Server-Side Technologies

  • Maven - The most popular Java build tool
  • Spring Boot - Spring is the de-facto standard for Java apps
  • Liquibase - Database updates made easy (good schema version control)
  • JPA - The standard ORM solution for Java
  • Spring Data JPA - Removes a lot of boilerplate code
  • Caching - Ehcache or HazelCast including clustering support
  • Spring Security - Handles cookie authentication and OAuth2
  • Spring MVC REST - Add a few annotations and you are done
  • Monitoring - Built-in metrics

Another code generator? <blech>

  • No strange hard to read code
  • No strange custom extension files with metadata
  • Uses all standard, open source frameworks
  • However, you need to know (or get to know) the frameworks being used!

Go as far as you want

  • Take the scaffold as a starting point and run with it
  • Use JHipster to add entities and services
  • Try out Docker?

Future of JHipster
